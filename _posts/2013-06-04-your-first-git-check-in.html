In the last post we created our first python project in Eclipse by cloning a Git Project from Github.   Since we cloned a project from Github this gave us two things:

  • Local Git Repository which is an copy/clone of the original project
  • The ability to “Push” our changes back to the remote Github project

You don’t have to “push” your changes back to the remote repository.  But this is a great way to keep you code in sync is you are working from different computers (e.g. work desktop/home laptop).  Obviously you have to push your change to the remote server if the project involves more than one developer.

In this example we will make a minor change to the views.py file by changing the “Hello” text to “Hello Git!” while in the Resource or PyDev Perspective. Notice that once you make a change to the file there is a “>” symbol beside views.py. This indicates that the file has changed.

Open views.py  file under mysite. Change the text.

Now the changes can be committed.  Right click on the views.py file as shown below:

Team > Commit Type in a “commit” message”

Click “Commit” and your changes have been checked-in to the local git repository.  Right click the views.py file again and select Team > Show in History.  This will display the change history for the file.

In a future post the changes will be pushed to the remote repository.
